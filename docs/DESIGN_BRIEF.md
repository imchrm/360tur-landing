# DESIGN BRIEF — 360tur-landing

Бриф для дизайн-агента: структурированное описание аудитории, бренда, требований к лайауту и интерактиву лендинга.

**Статус:** заполнен (все 4 фазы пройдены).

---

## 1. Бизнес и аудитория

### 1.1 Целевая аудитория
Основные сегменты B2B/B2B2C:
- Агентства недвижимости и застройщики
- Отели, хостелы, апартаменты (hospitality)
- Рестораны, кафе, клубы (HoReCa)
- Музеи, галереи, выставочные пространства
- Образовательные учреждения (школы, вузы)
- Магазины, шоу-румы, автосалоны

Дизайн должен универсально «цеплять» эти сегменты без узкой стилистики под один из них (т.е. не чисто «люксовая недвижимость» и не чисто «музейная эстетика»).

### 1.2 География
- Ташкент (основной фокус)
- Весь Узбекистан (расширенный охват)

Подача — локальная/региональная, можно использовать местные визуальные акценты, но без излишней этники.

### 1.3 Основные цели (KPI)
Три равнозначных призыва к действию:
1. **Позвонить** — номер должен быть виден и кликабелен (`tel:`).
2. **Написать в мессенджер** — Telegram, WhatsApp, Facebook Messenger (прямые ссылки / кнопки).
3. **Посмотреть портфолио** — ссылка на внешний сайт `360tur.uz`.

В дизайне необходимо **3 параллельных CTA**, не конкурирующих между собой (например, группой кнопок «Позвонить / Написать / Портфолио» или через sticky-панель).

### 1.4 Вторичные цели
- Повышение узнаваемости бренда.
- SEO-трафик (структура контента и заголовков должна быть SEO-дружественной: H1/H2, семантика, alt-тексты).

### 1.5 Конкурентные преимущества (для тизеров/блоков «Почему мы»)
1. **Скорость** выполнения.
2. **Цена** / «качество за разумные деньги».
3. **Собственный хостинг туров** (клиент получает готовую ссылку, не платит отдельно за хостинг).
4. **Локальное присутствие** (Ташкент / Узбекистан — оперативный выезд, коммуникация на узбекском/русском).

### 1.6 Ценовое позиционирование
**«Качество за разумные деньги»** — не премиум, не «самое дешёвое». Визуал должен передавать профессионализм и надёжность без ощущения «люкса» и без «бюджетности».

### 1.7 Источники трафика (влияет на first-screen и мобильную адаптацию)
- Соцсети (Instagram, Facebook, Telegram) → ссылки в bio/stories → **мобильный трафик доминирует**, first-screen должен работать на телефоне в вертикальной ориентации.
- SEO-трафик → важен текстовый контент и его разметка.
- Визитки и QR → офлайн переходы, часто первое знакомство — мобильное.

**Вывод:** mobile-first обязателен; desktop — адаптивная версия, а не наоборот.

## 2. Бренд и визуальный стиль

### 2.1 Логотип и фирменный стиль
- **Логотип:** есть в формате SVG.
- **Брендбук:** отсутствует — правила оформления (типографика, сетка, иконки, применение цвета) **должны быть выработаны вместе с дизайн-агентом** в рамках этого проекта.
- **Фиксированных шрифтов и цветов до проекта не было** — палитра задана ниже (п. 2.3), типографика — в п. 2.4.

### 2.2 Стилистическое направление
**Чистое / минималистичное:** белый фон, тёмный текст, один-два акцентных цвета. Без перегруженности, без градиентных «цифровых» фонов.

### 2.3 Цветовая палитра

| Роль | HEX | Комментарий |
|------|------|-------------|
| Основной акцент (Primary) | `#5C092C` | Тёмный бордо (burgundy) |
| Вторичный A (Secondary A) | `#6C1C0B` | Тёмный винно-коричневый |
| Вторичный B (Secondary B) | `#3B0847` | Глубокий фиолетовый |
| Фон (Background) | `#FFFFFF` | Белый |
| Основной текст (Text) | `#131313` | Почти чёрный |

Палитра выстроена на контрасте «белый фон × глубокие насыщенные акценты» — передаёт премиальность и сдержанность без «люксовой» показухи. Дизайн-агент может предложить дополнительный нейтральный/серый и системные цвета (success / warning / error) под стиль палитры.

### 2.4 Типографика
**Современный гротеск** — на выбор дизайнера из семейства: `Inter`, `Manrope`, `Geist`, `Golos Text` и подобных.
Требование: шрифт должен поддерживать **кириллицу и латиницу** (для `uz` используются обе, `ru` — кириллица, `en` — латиница).

### 2.5 Референсы
Готовых референсов и анти-референсов пока нет — подбор визуальных референсов остаётся за дизайн-агентом; они должны соответствовать направлению 2.2 и палитре 2.3.

### 2.6 Tone of voice
Нейтральный, на «вы», лёгкий, **без канцелярита**. Без эмодзи в интерфейсных текстах.

### 2.7 Фото и иллюстрации
- **Скриншоты / превью готовых туров** — есть (можно использовать в блоках портфолио, «Как работает», Hero).
- Фото процесса съёмки, оборудования, команды — **отсутствуют**, в дизайне не использовать.
- **Иконки — плоские** (flat, single-color / two-color, в рамках палитры п. 2.3).

### 2.8 Эмоциональные ассоциации
Сайт должен восприниматься как: **технологичный, профессиональный, премиальный, надёжный, современный**.

## 3. Структура и контент секций

### 3.1 Header + Navigation
- **Элементы:** логотип, меню-якоря (`About`, `Services`, `How`), `LanguageSwitcher`, CTA-кнопка «Контакты».
- **CTA «Контакты / Связаться»:** при наведении (hover) раскрывается меню-блок с телефоном и кнопками мессенджеров. На мобильных — поведение по `tap` (hover отсутствует).
- **Sticky-хедер** и стиль мобильного меню — _на усмотрение дизайн-агента в рамках общего визуального стиля_ (требуется предложение с обоснованием).

### 3.2 Hero (первый экран)
- **H1 (оффер):** «Виртуальные 3D-туры для вашего бизнеса».
- **Подзаголовок:** «Профессионально, быстро и недорого».
- **Визуал:** **встроенный интерактивный 360°-просмотрщик** (drag-крутится), используется пример реального тура.
- **CTA в Hero:**
  - Кнопка «Позвонить» (`tel:`)
  - Кнопка «Портфолио» (ведёт на `360tur.uz`)
  - Иконки мессенджеров (Telegram, WhatsApp, Facebook Messenger)

> ⚠️ **Замечание по позиционированию:** слово «недорого» в подзаголовке слабо сочетается с эмоциональными ассоциациями «премиальный / профессиональный» (п. 2.8). Дизайн-агенту предложить 2–3 альтернативные формулировки (например, «Профессионально, быстро и по разумной цене» / «Быстро, качественно, без переплат»).

### 3.3 About / AboutCards (О нас)
- **About:** краткая миссия компании (1–2 абзаца, копирайтер).
- **Ключевые цифры (счётчики):** количество довольных клиентов, количество сегментов (отраслей).
- **AboutCards — «Почему мы»:** 4 карточки с преимуществами:
  1. **Скорость** (быстрый выезд и сдача)
  2. **Цена** (разумная стоимость без переплат)
  3. **Хостинг** (размещение туров включено)
  4. **Локальность** (присутствие в Ташкенте / по Узбекистану)

### 3.4 Services (Услуги)
Комбинированный блок, 2 уровня:
1. **Верхний блок — по типам услуг** (карточки): съёмка 360° / сборка тура / хостинг / брендирование / интеграция на сайт клиента. _Точный перечень типов услуг уточнить с клиентом на этапе копирайтинга._
2. **Нижний блок — по отраслям** (карточки с мини-кейсами): недвижимость, отели/хостелы, HoReCa, музеи/галереи, образование, ритейл/шоурумы/автосалоны (из п. 1.1).

Формат отображения: **карточки** (без аккордеонов / таблиц).

### 3.5 HowWeWork (Процесс)
4 шага:
1. **Заявка / связь / созвон** — обсуждение объекта и задач.
2. **Выезд и съёмка** — работа с 360°-камерой на объекте.
3. **Обработка и сборка тура** — постобработка, сборка виртуального тура.
4. **Размещение на хостинге, передача ссылки** — готовый тур доступен клиенту по ссылке.

Формат: горизонтальная или вертикальная временная шкала с номерами шагов, иконками и короткими описаниями.

### 3.6 Портфолио / Примеры работ
- **Встроенный iframe** одного демо-тура прямо на странице лендинга.
- Обоснование выбора iframe: хостинг туров находится на той же инфраструктуре, что и лендинг (`360tur.uz`) — нет cross-origin / политик доверия.
- Под iframe — кнопка «Все работы» → ведёт на `https://360tur.uz`.
- На первом этапе — **один кейс**. Позже возможно добавить сетку превью.

### 3.7 Testimonials (Отзывы)
Реальных отзывов пока нет. **В дизайне предусмотреть секцию-плейсхолдер**:
- Сетка карточек (3–6) с местом под аватар/логотип клиента, цитату, имя/должность.
- Если на момент публикации отзывов ещё нет — секцию временно скрыть, оставив готовый макет для быстрого наполнения в будущем.

### 3.8 FAQ (Частые вопросы)
Начальный набор вопросов (копирайтер напишет ответы):
1. Сколько стоит тур?
2. Сколько времени занимает съёмка?
3. Что если объект большой / многоэтажный?
4. Включено ли размещение (хостинг) в цену?
5. Можно ли встроить тур на сайт клиента?

Формат: аккордеон (одиночное раскрытие).

### 3.9 LeadForm / Contact (Заявка и контакты)
- **Формы заявки на текущем этапе не будет.** В дизайне — **плейсхолдер-блок** «Связаться с нами» с теми же CTA, что в Hero и в хедере (позвонить / мессенджеры / портфолио). Макет формы подготовить отложено — после решения открытого вопроса из `TODO.md`.
- **Контакты на странице:**
  - Телефон (кликабельный, `tel:`)
  - Мессенджеры: Telegram, WhatsApp, Facebook Messenger
  - Ссылки на соцсети — _уточнить перечень с клиентом_ (Instagram / Facebook / Telegram-канал и т.п.)
  - Email, физический адрес офиса — _уточнить с клиентом, отображать ли_

### 3.10 Footer
- **Контакты** (телефон, мессенджеры) + **соцсети**.
- **Ссылки на политику конфиденциальности и условия** (`/privacy`, `/terms`) — тексты готовятся отдельно.
- Логотип, копирайт-строка, дублирование `LanguageSwitcher` — _на усмотрение дизайн-агента_.

### 3.11 Маршруты и удаления относительно шаблона
- **Удалить страницы:** `ConstructionWaste.jsx`, `HouseholdWaste.jsx` (остатки шаблона вывоза мусора).
- **Активные маршруты:**
  - `/` — главная (единая длинная страница со всеми секциями выше)
  - `/privacy` — политика конфиденциальности (опционально)
  - `/terms` — условия использования (опционально)
- Блог, калькулятор стоимости, онлайн-чат — не включать в MVP.

## 4. Интерактив, анимации, приоритеты, SEO

### 4.1 Анимации и движение
**Минимум движения:**
- `fade-in` / `slide-up` при скролле (IntersectionObserver).
- Плавные hover-эффекты для интерактивных элементов (кнопки, ссылки, карточки).
- Без параллакса, сложных canvas/WebGL-фонов и scroll-driven эффектов.
- Обязательное уважение к `prefers-reduced-motion: reduce` — при активации отключать все анимации кроме мгновенных transitions.

### 4.2 Интерактивный 360°-просмотрщик в Hero
- **Плеер:** `Marzipano` (уже используется на `360tur.uz`). Так как лендинг и туры размещаются на одном домене (`360tur.uz`), встраивание — либо через `iframe`, либо через прямую инициализацию Marzipano в `div`.
- **Ключевое требование к дизайну:** плеер должен оставаться **в рамках ограниченного контейнера Hero-секции** (фиксированные пропорции, например, `16:9` или `21:9`), не растягиваясь на весь экран и не «выпрыгивая» за пределы секции при взаимодействии.
- **Fallback для слабых устройств / мобильных:** не требуется — плеер грузится сразу.

### 4.3 Производительность и Core Web Vitals
- **Цель:** мобильный **Lighthouse Performance ≥ 80**.
- Следствия для дизайна:
  - Шрифты: не более 2 начертаний, самозагрузка через `@font-face` с `font-display: swap`.
  - Картинки: оптимизированные форматы (WebP / AVIF), `<picture>` с source-set для ретины, lazy-loading для не-above-the-fold.
  - 360°-плеер: загрузка отложена до появления Hero-секции в viewport (по возможности).

### 4.4 Доступность (a11y)
- **Цель:** соответствие **WCAG 2.1 AA** (минимальный уровень).
- **Обязательный аудит контрастности** палитры:
  - `#131313` на `#FFFFFF` — проходит (AAA).
  - Белый текст на `#5C092C` / `#6C1C0B` / `#3B0847` — требует проверки (обычно проходит AA для крупного текста, для мелкого текста на `#6C1C0B` может не проходить — проверить).
- Стандартные требования: навигация с клавиатуры, видимые focus-states, `alt` для изображений, семантические теги, ARIA-атрибуты на интерактивных элементах. По минимуму — без углублённого скрин-ридер-тестирования.

### 4.5 SEO
- **Технические файлы:** `sitemap.xml`, `robots.txt` — подготовить.
- **Мета-теги:** на каждую локаль (`ru`, `uz`, `en`) с корректными `<link rel="alternate" hreflang="…">` и `x-default`.
- **Open Graph / Twitter Cards:** дизайнеру подготовить **OG-image-макет `1200×630`** (основной оффер + логотип + скриншот тура).
- **Микроразметка Schema.org:** `LocalBusiness` с полями `name`, `address`, `telephone`, `openingHours`, `url`, `image`, `sameAs` (соцсети). Текстовые значения заполняются на этапе копирайтинга.

### 4.6 Мультиязычность — поведение по умолчанию
- **Определение языка при первом входе:** по `navigator.language` / `Accept-Language` (приоритет: `ru` → `uz` → `en`; fallback — `ru`).
- Ручное переключение — через `LanguageSwitcher` в хедере/футере; выбор сохраняется (`localStorage`) и имеет приоритет над автоопределением при повторных визитах.
- К старту готовы **все три локали** (ru / uz / en).

### 4.7 Аналитика и трекинг
- **Yandex.Metrica** (актуально для СНГ-трафика).
- **Google Analytics 4**.
- **Отслеживание кликов по CTA** (позвонить / мессенджеры / портфолио) — **не нужно** на текущем этапе (достаточно page-view метрик).
- Скрипты аналитики загружать после основного контента (не блокировать рендер).

### 4.8 Тёмная тема
**Не требуется.** Единая светлая тема (белый фон, глубокие акценты).

### 4.9 Приоритеты секций
Все секции релиза **одинаково важны**, выделения MVP-блоков не требуется — делать все секции к первой публикации.

### 4.10 Формат передачи и сроки
- **Формат поставки:** **готовый React-проект** (Vite + Tailwind + React Router + React Context для i18n), а не только Figma-макеты. Дизайн-агент — full-stack по смыслу: и визуал, и код в рамках уже зафиксированной архитектуры (см. `docs/ARCHITECTURE.md`).
- **Отдельные макеты под разные языки не нужны** — один макет с проверкой на длину строк основных языков (`ru`, `uz`, `en`).
- **Жёстких сроков нет**, работу вести по договорённости.

## 5. Итоги для дизайн-агента (чеклист)

### Техническая среда (уже зафиксирована)
- Стек: **React 18 + Vite + Tailwind CSS + React Router (`BrowserRouter`)**.
- i18n: **React Context + плоские словари `src/locales/{ru,uz,en}.js`** (без `i18next`).
- State: **`useState` + React Context API**.
- Ассеты: импортируемые — в `src/assets/`, статические — в `public/`.
- Хостинг: статический на `360tur.uz` (SPA-fallback настраивается на стороне хостинга).
- Подробнее — `docs/ARCHITECTURE.md`.

### Компоненты (на базе шаблона `musor/src/components/`, требуют полной переработки визуала и частично — логики)
- `Header.jsx` + `Navigation.jsx` + `LanguageSwitcher.jsx`
- `Hero.jsx` (+ интеграция Marzipano)
- `About.jsx` + `AboutCards.jsx`
- `Services.jsx`
- `HowWeWork.jsx`
- Портфолио (встроенный iframe — **новый компонент или блок в рамках `Hero`/отдельной секции**, в шаблоне такого нет)
- `Testimonials.jsx` (плейсхолдер)
- `FAQ.jsx`
- `Contact.jsx` + `ContactInfo.jsx` (без `LeadForm` на первом этапе — плейсхолдер)
- `Footer.jsx`
- **Удалить:** `LeadForm.jsx` (как активный компонент — до момента решения вопроса о форме), страницы `ConstructionWaste.jsx`, `HouseholdWaste.jsx`, вся логика Telegram-бота из `Contact.jsx`.

# ARCHITECTURE — 360tur-landing

Документ фиксирует архитектурные решения и их обоснования.
Обновляется при значимых изменениях в структуре проекта.

---

## Сборщик

**Vite** (не Create React App)

Причина: шаблон уже использует Vite. CRA устарел и не поддерживается.
Конфигурация: `musor/vite.config.js`

## CSS

**Tailwind CSS** через PostCSS.
Конфигурация: `musor/tailwind.config.js`
Кастомные токены (цвета, шрифты) — только там, не в компонентах напрямую.

## Маршрутизация

**React Router**. Лендинг одностраничный, но роутер оставлен на случай
появления отдельных страниц (условия, политика и т.д.).

## Компоненты

Flat-структура в `src/components/` — без вложенных папок, пока компонентов мало.
При росте числа компонентов — группировать по фиче: `components/hero/`, `components/contact/`.

## Решения
